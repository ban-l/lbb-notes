    HikariCP连接池

    HikariCP 是一个高性能的JDBC连接池，具有许多可配置的参数。以下是HikariCP的一些常用参数及其默认值：

    1. maximumPoolSize：默认值为10。
      • 连接池中允许的最大连接数。
    2. minimumIdle：默认值为maximumPoolSize的值。
      • 连接池中保持空闲状态的最小连接数。
    3. idleTimeout：默认值为600000（10分钟）。
      • 连接在池中空闲的最长时间，超过这个时间将被释放。如果该值为0，则禁用空闲连接超时。
    4. maxLifetime：默认值为1800000（30分钟）。
      • 连接的最长生命周期，超过这个时间的连接将被释放并替换。如果该值为0，则禁用最大生命周期。
    5. connectionTimeout：默认值为30000（30秒）。
      • 从池中获取连接的最大等待时间，如果超时将抛出SQLException。
    6. validationTimeout：默认值为5000（5秒）。
      • 连接验证操作的最大时间。
    7. leakDetectionThreshold：默认值为0（禁用）。
      • 连接泄漏检测的阈值时间。如果连接未在指定时间内关闭，将记录警告日志。
    8. autoCommit：默认值为true。
      • 控制连接的默认自动提交行为。
    9. poolName：默认值为自动生成的名称。
      • 连接池的名称，通常用于日志和监控。

    配置示例

    application.properties

    spring.datasource.hikari.minimum-idle=10
spring.datasource.hikari.maximum-pool-size=10

    application.yml

    spring:
  datasource:
    hikari:
      minimum-idle: 10
      maximum-pool-size: 10

    代码示例

    在 Java 代码中配置 HikariCP：

    import com.zaxxer.hikari.HikariConfig;
import com.zaxxer.hikari.HikariDataSource;

public class DataSourceConfig {

    public static void main(String[] args) {
        HikariConfig config = new HikariConfig();
        config.setJdbcUrl("jdbc:mysql://localhost:3306/mydb");
        config.setUsername("username");
        config.setPassword("password");

        // 设置最小空闲连接数
        config.setMinimumIdle(10);

        // 设置最大连接池大小
        config.setMaximumPoolSize(10);

        HikariDataSource dataSource = new HikariDataSource(config);

        // 使用 dataSource 进行数据库操作
    }
}

    总结

    HikariCP 的默认最小空闲连接数和最大连接池大小都是 10。根据需求调整这些参数以优化性能。
