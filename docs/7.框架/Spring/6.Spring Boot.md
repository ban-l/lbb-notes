# Spring Boot

## 概念

Spring 开源组织下的子项目，Spring 组件一站式解决方案。

目的是简化 Spring 使用难度，简化配置，提供了各种启动器，开发者能快速上手（约定大于配置，化繁为简）。

1. 简化Spring应用初始搭建以及开发过程，使用特定方式来进行配置；

1. 创建独立的Spring引用程序main方法运行；
2. 嵌入tomcat，无需部署war文件；

1. 简化maven配置；
2. 自动配置。

## Spring问题

- 配置重复添加，如spring-core、spring-bean、spring-context等；

- 框架使用、环境参数重复配置，如开启注解、配置日志等。

## SpringBoot好处

### 管理依赖关系

创建Spring Boot Starter 项目时，按需选择功能，Spring Boot自动管理依赖关系。

### 独立运行

Spring Boot 而且内嵌了各种 servlet 容器，Tomcat、Jetty 等，不再需要打成 war 包部署到容器中，Spring Boot 项目打包成一个可执行的 jar 包，可独立运行（Java-jar命令），所有的依赖包都在一个 jar 包内。

### 简化配置

启动器自动依赖其他组件，减少了 maven 的配置，例如 spring-boot-starter-web。

### 自动配置

使用基于类路径和应用程序上下文的智能默认值。

Spring Boot 能根据当前类路径下的类、jar 包来自动配置 bean，如添加一个 springboot-starter-web 启动器，就能拥有 web 的功能，无需其他配置。

### 无代码生成和XML配置（注解）

Spring Boot 配置过程中无代码生成，也无需 XML 配置文件就能完成所有配置工作，这一切都是借助于条件注解完成的。

### 应用监控

Spring Boot 提供一系列端点可以监控应用。

## Spring、Spring MVC、SpringBoot

### Spring

当使用DI或者是IOC的时候，可以开发松耦合应用。

### Spring MVC

Spring MVC提供了一种分离式的方法来开发Web应用，使 Web 开发变的非常简单。

### SpringBoot

Spring和Spring MVC的问题在于需要配置大量的参数。

SpringBoot通过自动配置和启动来解决这个问题。
